Intro

회귀모형을 더 잘 만드는 세 가지 축

회귀분석에서는 단순히 선형회귀를 한 번 적합하는 것에서 끝나지 않습니다. 실제 분석에서는 어떤 변수를 남길지, 과적합을 어떻게 줄일지, 비선형 관계를 어떻게 잡을지가 매우 중요합니다.

1. 변수선택법

예측에 꼭 필요한 변수만 남겨 모델을 단순화하고 해석력을 높입니다.

2. 정규화 회귀

계수 크기에 패널티를 주어 과적합을 줄이고 일반화 성능을 높입니다.

3. 트리 기반 회귀

비선형 관계와 변수 간 상호작용을 자동으로 반영할 수 있습니다.

핵심 관점

해석예측복잡도 제어를 균형 있게 보는 것이 중요합니다.

Variable Selection

변수선택법이 필요한 이유

변수 선택(variable selection)의 목적은 예측에 꼭 필요한 변수만 남겨서 모델을 단순하게 만들고, 동시에 과적합을 줄이며 해석력을 높이는 것입니다.

목적설명
불필요한 변수 제거의미 없는 변수를 제외하여 모델을 깔끔하게 만듭니다.
모델 단순화변수 수를 줄이면 해석이 쉬워집니다.
과적합 방지훈련데이터에만 지나치게 맞는 현상을 줄일 수 있습니다.
해석 용이성 향상어떤 변수가 중요한지 더 명확하게 설명할 수 있습니다.
예측 성능 개선잡음 변수를 제거하면 테스트 성능이 좋아질 수 있습니다.
변수가 많다고 항상 좋은 모델은 아닙니다. 설명력은 조금 높아져도 일반화 성능은 오히려 나빠질 수 있습니다.
Selection Methods

변수선택법의 대표적인 종류

전진 선택법 (Forward Selection)

  • 변수가 없는 상태에서 시작
  • 가장 성능이 좋아지는 변수를 하나씩 추가
  • 계산량이 비교적 적음
  • 초기에 잘못 들어간 변수를 뒤늦게 빼기 어려움

후진 제거법 (Backward Elimination)

  • 모든 변수를 넣고 시작
  • 덜 중요한 변수를 하나씩 제거
  • 전체 변수를 고려해서 시작한다는 장점
  • 변수가 많으면 계산량이 큼

단계적 선택법 (Stepwise Selection)

  • 전진 선택과 후진 제거를 결합
  • 변수를 추가하기도 하고 제거하기도 함
  • 더 유연하지만 절차가 복잡해질 수 있음

전체 부분집합 선택법 (Best Subset Selection)

  • 가능한 모든 변수 조합을 비교
  • 가장 좋은 조합을 선택
  • 이론적으로 가장 철저함
  • 변수가 많아지면 계산량이 폭발적으로 증가
Selection Criteria

변수 선택 시 어떤 기준으로 판단할까?

변수 선택에서는 단순히 설명력이 높은 모델보다 설명력과 복잡도를 함께 고려한 모델을 선택하는 것이 중요합니다.

기준핵심 아이디어해석 포인트
AIC적합도 + 변수 수에 대한 페널티작을수록 좋음
BICAIC와 비슷하지만 변수 수에 더 강한 페널티작을수록 좋음
Mallows' Cp편향과 분산의 균형 확인작고 단순한 모델 선호
교차검증 점수훈련셋이 아닌 검증 성능을 직접 확인예측 중심 분석에서 중요

즉, 좋은 모델은 잘 맞기만 하는 모델이 아니라, 필요 이상으로 복잡하지 않은 모델입니다.

AIC · BIC · Cp

AIC, BIC, Mallows' Cp 핵심 정리

1. AIC (Akaike Information Criterion)

AIC = -2 log L + 2k
  • L: 가능도(likelihood), k: 모수 개수
  • 데이터에 잘 맞을수록 좋지만, 변수가 많아질수록 벌점을 줍니다.
  • AIC 값이 작을수록 좋은 모델입니다.
  • 상대적으로 예측 성능 중심의 모델 선택에 자주 사용됩니다.

2. BIC (Bayesian Information Criterion)

BIC = -2 log L + k log n
  • 표본 수 n이 커질수록 벌점이 더 커집니다.
  • AIC보다 변수 수에 대해 더 보수적입니다.
  • 단순한 모델을 더 선호하는 경향이 있습니다.

3. Mallows' Cp

  • 모형의 편향과 분산 사이 균형을 보려는 지표입니다.
  • 값이 너무 크면 과적합 또는 부적절한 변수 조합일 수 있습니다.
  • 후보 모델들을 비교할 때 참고 지표로 사용합니다.
지표특징언제 유용한가
AIC복잡도 벌점이 비교적 약함예측 성능을 좀 더 중시할 때
BIC복잡도 벌점이 더 강함더 단순한 모델을 선호할 때
Mallows' Cp편향-분산 균형 관점후보 모델 비교 보조 지표
Regularized Regression

정규화 회귀(Regularized Regression)란?

정규화 회귀는 회귀모형에서 변수의 계수(coefficient)가 너무 커지는 것을 방지하기 위해 규제(패널티)를 추가하는 방법입니다. 이를 통해 과적합을 줄이고 모델의 일반화 성능을 높일 수 있습니다.

기본 선형회귀: minimize Σ(yi - ŷi)² 정규화 회귀: minimize [ Σ(yi - ŷi)² + Penalty ]

대표적인 방법은 L1 규제(Lasso), L2 규제(Ridge), Elastic Net입니다.

정규화 회귀는 특히 변수가 많거나, 변수끼리 상관이 높거나, 과적합이 의심될 때 매우 유용합니다.
L1 Penalty

L1 규제: Lasso Regression

Penalty = Σ |βj|

L1 규제는 회귀계수의 절대값 합을 패널티로 사용하는 방법입니다.

핵심 특징

  • 일부 변수의 계수를 정확히 0으로 만들 수 있습니다.
  • 따라서 자동 변수선택 효과가 있습니다.
  • 중요한 변수만 남기고 싶을 때 유리합니다.

언제 적합한가?

  • 변수가 많고 불필요한 변수가 섞여 있을 때
  • 해석 가능한 단순 모델이 필요할 때
  • 변수선택을 동시에 하고 싶을 때
한 줄 요약: Lasso는 불필요한 변수를 제거하는 데 강한 규제 방법입니다.
L2 Penalty

L2 규제: Ridge Regression

Penalty = Σ βj²

L2 규제는 회귀계수의 제곱합을 패널티로 사용하는 방법입니다.

핵심 특징

  • 계수를 0에 가깝게 줄이지만 완전히 0으로 만들지는 않는 경우가 많습니다.
  • 모든 변수를 유지하면서도 계수의 크기를 안정화합니다.
  • 다중공선성이 있을 때 특히 유리합니다.

언제 적합한가?

  • 변수들이 모두 어느 정도 의미가 있다고 생각될 때
  • 변수 제거보다는 계수 안정화가 중요할 때
  • 상관이 높은 변수들이 많은 데이터일 때
한 줄 요약: Ridge는 변수를 없애기보다 계수를 작게 만들어 과적합을 줄입니다.
Mixed Penalty

Elastic Net: L1과 L2의 결합

Elastic Net은 L1과 L2 규제를 함께 사용하는 방법입니다.

Penalty = λ [ α Σ|βj| + (1-α) Σβj² ]
  • α = 1 이면 Lasso에 가까움
  • α = 0 이면 Ridge에 가까움
  • L1의 변수선택 기능과 L2의 안정성을 동시에 활용할 수 있습니다.

언제 적합한가?

  • 변수 수가 많고 상관이 높은 변수도 많은 경우
  • Lasso 하나로는 선택이 불안정할 때
  • 변수선택과 계수 안정화를 함께 원할 때
한 줄 요약: Elastic Net은 Lasso와 Ridge의 절충안입니다.
Comparison

L1 · L2 · Elastic Net 비교

방법패널티변수 제거주요 장점주의점
LassoL1가능자동 변수선택상관 높은 변수들 사이에서 선택이 불안정할 수 있음
RidgeL2거의 없음계수 안정화, 다중공선성 완화변수 수를 직접 줄이지는 않음
Elastic NetL1 + L2가능선택 + 안정성 동시 확보튜닝할 하이퍼파라미터가 더 많음

실무 감각으로 기억하기

  • 해석 가능한 간단한 모델이 필요하면 → Lasso
  • 다중공선성이 심하고 모든 변수를 유지하고 싶으면 → Ridge
  • 둘 다 필요하면 → Elastic Net
Tree Regression

트리 기반 회귀란?

트리 기반 회귀는 데이터를 여러 구간으로 나누면서 각 구간에서 예측값을 만드는 방법입니다. 선형회귀처럼 하나의 직선을 맞추는 것이 아니라, 조건 분기(rule)를 통해 비선형 패턴을 잡습니다.

핵심 아이디어

“어떤 변수의 어떤 기준값을 기준으로 나누면 오차가 가장 많이 줄어드는가?”를 반복합니다.

왜 강한가?

비선형 관계, 구간별 효과, 변수 간 상호작용을 자동으로 반영할 수 있습니다.

트리 회귀의 장점

  • 스케일링이 거의 필요 없습니다.
  • 비선형 관계를 잘 반영합니다.
  • 설명 규칙을 나무 구조로 볼 수 있어 직관적입니다.

트리 회귀의 단점

  • 단일 트리는 과적합되기 쉽습니다.
  • 데이터가 조금만 달라져도 구조가 바뀔 수 있습니다.
  • 예측 경계가 계단형으로 나타날 수 있습니다.
Tree Family

대표적인 트리 기반 회귀 모델

모델개념특징
Decision Tree Regressor하나의 트리로 예측직관적이지만 과적합 위험이 큼
Random Forest Regressor여러 트리를 무작위로 만들어 평균안정적이고 기본 성능이 좋음
Gradient Boosting Regressor이전 오차를 보완하는 트리를 순차적으로 추가정교한 예측 가능
XGBoost / LightGBM / CatBoost고성능 부스팅 계열실무·대회에서 매우 자주 사용

단일 트리 vs 앙상블

  • 단일 트리는 해석이 쉽지만 불안정할 수 있습니다.
  • 앙상블은 여러 트리를 결합하여 성능과 안정성을 높입니다.
Boosting Models

XGBoost · LightGBM · CatBoost 이론 정리

XGBoost

  • Gradient Boosting을 효율적으로 개선한 모델
  • 정규화, 병렬 처리, 가지치기 등이 잘 설계되어 있음
  • 정확도와 안정성 때문에 매우 널리 사용됨

LightGBM

  • 속도와 메모리 효율에 강점이 있는 부스팅 모델
  • 대용량 데이터에서 빠르게 학습 가능
  • 리프 중심 분할 방식으로 깊고 효율적인 트리를 만듦

CatBoost

  • 범주형 변수 처리에 강한 부스팅 모델
  • 별도의 복잡한 인코딩 없이도 좋은 성능을 내는 경우가 많음
  • 카테고리 feature가 많은 데이터에서 특히 강점
모델강점특히 유리한 상황
XGBoost정확도와 안정성전반적으로 강력한 기본 선택지
LightGBM속도와 대용량 처리행 수가 많고 빠른 반복 실험이 필요할 때
CatBoost범주형 처리카테고리 feature가 많을 때
Guide

언제 어떤 회귀 방법을 선택할까?

상황추천 방법이유
해석이 가장 중요함선형회귀 + 변수선택계수 해석이 명확하고 설명하기 쉬움
과적합이 걱정됨Ridge / Elastic Net계수 크기를 조절해 일반화 성능 향상
변수 수가 많고 불필요한 변수도 많음Lasso / Elastic Net변수선택 효과를 기대할 수 있음
비선형 관계가 많음트리 기반 회귀직선이 아닌 복잡한 패턴을 잘 잡음
최고 성능이 필요함XGBoost / LightGBM / CatBoost부스팅 계열이 높은 예측력을 보이는 경우가 많음

최종 정리

  • 변수선택법은 “무엇을 남길까”에 대한 문제입니다.
  • 정규화 회귀는 “계수를 얼마나 줄일까”에 대한 문제입니다.
  • 트리 기반 회귀는 “비선형 구조를 어떻게 잡을까”에 대한 문제입니다.
강의에서는 세 방법을 서로 경쟁 관계로 보기보다, 해석 중심 / 예측 중심 / 비선형 대응이라는 서로 다른 목적의 도구로 이해하는 것이 가장 좋습니다.
