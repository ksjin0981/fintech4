0. NLP 큰그림 (텍스트 → 숫자 → 모델 → 해석)

NLP는 텍스트를 그대로 “이해”하는 게 아니라, 텍스트를 숫자(벡터)로 바꾼 뒤 모델이 그 숫자 패턴을 학습하게 만드는 과정입니다. 그래서 NLP에서 가장 중요한 질문은 다음입니다.

  • 무슨 문제를 풀 건가? (분류/토픽/유사도/요약/생성)
  • 텍스트를 어떻게 숫자로 만들까? (BoW/TF-IDF/Embedding)
  • 어떤 모델이 그 숫자에서 패턴을 잘 찾나? (ML 모델 / DL 모델)

✅ 공통 파이프라인(실무 흐름)

1) 데이터 수집: 리뷰/상담로그/약관/공지/뉴스/사내 문서
2) 전처리: 정규화(소문자/특수문자), 토큰화, 불용어, 표제어/어간 추출(필요시)
3) 표현(벡터화): BoW/TF-IDF/Embedding(Word2Vec/문장임베딩)
4) 모델링: 분류/토픽(LDA)/유사도/요약/생성(Transformer)
5) 평가/운영: 정확도/F1, 토픽 해석, 모니터링, 재학습, KPI 대시보드
전통 ML vs DL 한줄
ML: BoW/TF-IDF 같은 “단어 기반 벡터” + 간단하고 빠른 모델 (해석 쉬움)
DL: Embedding + 신경망(RNN/LSTM/Transformer)로 “문맥/의미”를 더 깊게 학습 (성능↑)

5. 머신러닝(ML) 영역 — “단어 빈도 기반” 중심

ML 기반 NLP의 핵심은 BoW/TF-IDF/N-gram 같은 “단어 기반 벡터”를 만든 뒤 로지스틱/SVM/나이브베이즈 같은 모델로 분류하거나, LDA로 토픽을 찾는 것입니다.

ML이 잘 맞는 상황
빠르게 MVP, 비용 제약, 설명 가능성 필요(규제/리스크), 데이터가 많지 않을 때

5-1. BoW (Bag of Words) — “단어를 세어서 벡터로”

BoW는 문장의 단어 순서를 무시하고, 각 단어가 몇 번 등장했는지를 세어서 벡터로 만듭니다. 그래서 “단어 주머니”라고 부릅니다.

✅ 예시로 이해하기

문장1: "환불이 빨라요"
문장2: "환불이 늦어요"

Vocabulary = ["환불", "빠르", "늦"] (형태소/어간 기준으로 단순화)
BoW(문장1) = [1, 1, 0]
BoW(문장2) = [1, 0, 1]
장점
  • 단순/빠름, 적은 데이터에서도 강력한 베이스라인
  • 해석 쉬움(어떤 단어가 영향 줬는지 바로 보임)
단점
  • 단어 순서/문맥을 잃음(“하지만” 같은 반전 처리 약함)
  • 동의어 의미를 모름(“해지/탈퇴”를 따로 봄)
BoW가 의외로 잘 먹히는 이유

많은 텍스트 분류 문제는 “특정 단어/표현이 나오면 라벨이 결정되는 경우”가 많습니다. 예: “먹통/오류/로그인 안됨”이 나오면 불만/버그 라벨 가능성이 큼.

5-2. N-gram — “단어 묶음”으로 순서 정보를 조금 살리기

BoW는 순서를 완전히 버립니다. N-gram은 이를 보완하기 위해 연속된 N개의 토큰 묶음을 하나의 특징으로 사용합니다.

✅ 예시

문장: "로그인 오류가 계속 발생"
1-gram(유니그램): ["로그인","오류","계속","발생"]
2-gram(바이그램): ["로그인 오류","오류 계속","계속 발생"]
3-gram(트라이그램): ["로그인 오류 계속","오류 계속 발생"]
왜 중요?
“로그인”과 “오류”가 따로 있는 것보다 “로그인 오류”라는 묶음이 훨씬 강한 신호입니다.
핀테크에서는 “계좌 해지”, “비밀번호 오류”, “인증 실패” 같은 바이그램이 매우 유용합니다.
주의
N을 올리면 특징 수가 폭발합니다(차원↑). 그래서 보통 1~2 또는 1~3 정도로 제한하고, min_df, max_df로 단어를 정리합니다.

5-3. TF · IDF · TF-IDF — “중요한 단어에 가중치”

TF (Term Frequency)

한 문서 안에서 단어가 얼마나 자주 등장했는지(빈도)입니다.

IDF (Inverse Document Frequency)

전체 문서에 너무 흔한 단어는 정보량이 작으니 가중치를 낮춥니다. 예: “그리고/정말/하다” 같은 단어.

TF-IDF

TF × IDF로 계산하며, “문서 안에서는 자주 나오고, 전체에서는 흔하지 않은” 단어의 점수를 높입니다. 텍스트 분류/검색/키워드 추출에 매우 강력합니다.

핀테크 예
리뷰에서 “인증서/간편인증/OTP/먹통” 같은 단어는 특정 이슈를 강하게 대표합니다.
TF-IDF는 이런 “이슈 대표 단어”를 잘 부각시켜 줍니다.

5-4. WordCloud — “빈도/중요도를 눈으로 보기”

WordCloud는 단어를 크게/작게 표시하는 시각화입니다. 크기 기준은 보통 빈도(TF) 또는 TF-IDF 점수를 사용합니다.

언제 쓰나?
  • EDA(탐색)에서 데이터 분위기 파악
  • 긍정/부정 리뷰를 분리해 비교
  • 업데이트 전/후 키워드 변화 비교
함정
  • 많이 나온 단어 ≠ 중요한 단어
  • 불용어 제거 안 하면 의미 없는 단어가 크게 보임
  • 문맥(왜 부정인지)을 직접 보여주진 못함
실무 팁(퀄리티 올리기)
  • 불용어 사전을 직접 만들기(프로덕트 도메인 단어 포함)
  • 명사/형용사 중심으로 토큰을 제한하면 해석이 쉬움
  • 단순 빈도보다 TF-IDF 상위 단어로 만들면 “그룹 특화 단어”가 잘 보임

5-5. LDA 토픽 모델링 — “문서를 주제 비율로 분해”

LDA(Latent Dirichlet Allocation)는 대표적인 토픽 모델입니다. 핵심 가정은 다음 2가지입니다.

  • 토픽은 단어의 확률 분포이다. (토픽 → 단어 분포)
  • 문서는 토픽의 혼합이다. (문서 → 토픽 분포)

✅ 직관 예시

토픽 A(배송): ["배송","포장","빠르다","기사"] 확률이 높음
토픽 B(환불): ["환불","취소","지연","처리"] 확률이 높음

문서1: 배송 0.8 + 환불 0.2
문서2: 환불 0.9 + 배송 0.1

K(토픽 수) 선택이 핵심

K가 너무 작으면
여러 주제가 한 토픽에 섞여 “이름 붙이기”가 어려워집니다.
K가 너무 크면
토픽이 쪼개져서 서로 비슷한 토픽이 여러 개 생기고 해석이 어려워집니다.
핀테크/VOC에 LDA가 잘 맞는 이유
리뷰/상담로그는 “무슨 이슈가 많은지”를 빠르게 뽑는 게 목적일 때가 많습니다.
LDA는 “배송/환불/인증/오류/수수료” 같은 주제 지도를 빠르게 만들어줍니다.
한계
LDA는 BoW 기반이라 “문맥/순서”를 반영하지 못합니다. (그래도 탐색/설명가능성/속도에서 강점)

6. Embedding & Word2Vec — “의미를 담는 벡터”

BoW/TF-IDF는 의미를 직접 담지 못합니다. Embedding은 단어를 “의미 기반” 벡터로 만들고, 의미가 비슷한 단어는 벡터 공간에서도 가깝게 배치되도록 학습합니다.

직관
“사과”와 “바나나”가 가까이, “자동차”는 멀리. 이런 관계를 데이터로부터 자동 학습합니다.

Word2Vec이란?

Word2Vec은 “주변 단어(Context)가 비슷하면 의미도 비슷하다”라는 가정으로 단어 벡터를 학습합니다. 대표 구조가 CBOW와 Skip-gram입니다.

6-1. CBOW — “주변 단어로 중심 단어 맞히기”

CBOW(Continuous Bag of Words)는 주변 단어(Context)를 입력으로 넣고, 중심 단어(Target)를 예측합니다.

✅ 예시

문장: "나는 사과를 먹었다"
윈도우 크기=1이라면
Context: ["나는", "먹었다"] → Target: ["사과"]
특징
보통 학습이 빠르고 안정적인 편이라 “데이터가 많을 때” 잘 동작하는 경우가 많습니다.

6-2. Skip-gram — “중심 단어로 주변 단어 맞히기”

Skip-gram은 CBOW의 반대입니다. 중심 단어(Target)로 주변 단어(Context)를 예측합니다.

✅ 예시

문장: "나는 사과를 먹었다"
Target: ["사과"] → Context: ["나는", "먹었다"]
특징
희귀 단어(자주 안 나오는 단어) 학습에 상대적으로 유리하다고 알려져 있습니다.

6-3. t-SNE — “임베딩을 2D로 내려서 눈으로 확인”

단어 임베딩은 보통 100~300차원입니다. t-SNE는 이를 2차원으로 줄여서 “비슷한 단어가 실제로 모여 있는지” 시각적으로 확인하는 데 쓰입니다.

주의
t-SNE는 로컬 구조는 잘 보여주지만, 전역 거리(멀리 떨어진 정도)는 과해석하면 안 됩니다. 실행할 때마다 모양이 달라질 수 있으니 random_state 고정이 중요합니다.

7. 딥러닝(DL) 영역 — “문맥(순서/관계)”을 학습

DL은 Embedding을 입력으로 받아 문장/문서의 문맥을 학습합니다. 과거에는 RNN/LSTM이 중심이었고, 현재는 Attention/Transformer가 주류입니다.

7-1. RNN — 순서대로 읽으며 기억 전달

RNN은 문장을 왼쪽→오른쪽으로 읽으면서 “이전 요약(기억)”을 다음 시점에 전달합니다. 그래서 BoW/TF-IDF보다 순서 정보를 더 잘 다룰 수 있습니다.

한계
문장이 길어지면 앞부분 정보가 사라지는 문제가 있어 LSTM이 등장했습니다.

7-2. LSTM — “무엇을 기억/삭제할지” 게이트로 제어

LSTM은 게이트(Forget/Input/Output)로 메모리(셀 상태)를 관리해 긴 문맥에서도 정보가 덜 사라지도록 개선한 RNN 계열 모델입니다.

7-3. Attention — 중요한 단어에 집중

Attention은 문장을 해석할 때, 단어마다 중요도를 점수로 매겨 중요한 단어에 더 집중하도록 만드는 방법입니다. 번역/요약/질의응답에서 큰 효과를 냈습니다.

7-4. Transformer — Self-Attention 기반의 표준

Transformer는 RNN처럼 순서대로 처리하지 않고, 문장 전체를 한 번에 보고 Self-Attention으로 단어 간 관계를 학습합니다. 병렬처리가 가능하고 긴 문맥에도 강해서 현재 NLP의 표준이 됐습니다.

  • Self-Attention: 단어들이 서로에게 주는 영향 계산
  • Multi-Head: 관계를 여러 관점으로 나눠서 봄
  • Positional Encoding: 순서 정보 보완

8. ML vs DL 선택 기준

상황추천이유
빠른 MVP/비용 제약ML(TF-IDF + 선형모델)빠르고 해석 쉬움
문맥이 중요/표현 다양DL(Transformer)의미/뉘앙스에 강함
요약/생성/고급 검색DL(Transformer)생성형/의미 검색 적합
규제/설명가능성 중요ML 우선단어 가중치로 설명 쉬움
실무 황금 규칙
ML로 기준선 → 성능/문맥/요약 필요 시 DL로 업그레이드

9. 실무 체크리스트(운영 포함)

  • 라벨 정의: 환불/해지/인증오류 등 기준을 문장 예시로 명확히
  • 불용어/정규화: 워드클라우드/TF-IDF/LDA 품질에 결정적
  • 데이터 누수: “처리완료” 같은 미래정보가 학습에 들어가지 않게
  • 오탐/미탐 정책: 수동 검수/재학습/알림 기준
  • KPI: 토픽 비중 변화, 부정 비중, 처리시간 감소, 상담 이관율 감소 등

10. 최종 요약

ML 영역(단어 기반 벡터)

  • BoW, N-gram, TF-IDF
  • WordCloud(시각화), LDA(토픽)
  • 장점: 빠름/저비용/해석 쉬움

DL 영역(문맥/의미 학습)

  • Embedding(Word2Vec)
  • RNN/LSTM → Attention/Transformer
  • 장점: 문맥/뉘앙스/요약·생성에 강함
핀테크/기획 연결
ML로 VOC/리스크 텍스트를 빠르게 구조화(분류/토픽/키워드)하고,
DL로 상담요약/의미검색/고정밀 분류를 붙이면 서비스 기획 기능으로 자연스럽게 이어집니다.
