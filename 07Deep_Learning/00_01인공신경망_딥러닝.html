딥러닝은 데이터를 여러 층으로 반복 변환하면서 중요한 특징을 자동으로 찾아내고, 예측이 틀릴 때마다 오차를 줄이도록 가중치를 조금씩 수정하는 학습 방법입니다.

입력 데이터 → 계산 → 예측 → 오차 측정 → 가중치 수정 → 다시 계산 → 반복
핵심: 신경망 학습은 결국 “틀린 만큼 수정하는 반복 과정”입니다.

1. 퍼셉트론(Perceptron)

퍼셉트론은 가장 단순한 인공신경망입니다. 입력값을 받아 각각 중요도를 반영하고, 최종적으로 어느 쪽 클래스로 분류할지 결정합니다.

퍼셉트론의 기본 구조

입력 x₁, x₂, x₃ ... → 가중치 w₁, w₂, w₃를 곱함 → 모두 더함 → 편향 b를 더함 → 활성화 → 결과 출력
z = w₁x₁ + w₂x₂ + ... + wₙxₙ + b
  • x: 입력값
  • w: 각 입력의 중요도
  • b: 기준점 조정값(편향)
  • z: 최종 점수
그림 설명: 퍼셉트론은 기본적으로 직선 하나로 두 클래스를 나누는 모델입니다. 그래서 직선으로 나눌 수 있는 단순한 문제에는 잘 작동하지만, 복잡하게 꼬인 패턴에는 한계가 있습니다.

쉬운 비유

집값을 예측한다고 생각해보겠습니다. 면적, 위치, 층수, 교통 등 여러 정보가 들어옵니다. 이때 퍼셉트론은 각 항목에 중요도를 매긴 뒤, 모두 합쳐서 “좋은 집인지 아닌지”를 판단하는 아주 단순한 계산기라고 볼 수 있습니다.

XOR 예시: XOR 데이터는 직선 하나로 분리되지 않습니다. 이것이 퍼셉트론 하나로는 해결할 수 없는 대표적인 예입니다.
장점한계
구조가 단순해서 이해하기 쉽다복잡한 비선형 문제를 잘 못 푼다
신경망의 출발점을 이해하기 좋다XOR 같은 문제를 해결하지 못한다
중요: 퍼셉트론 하나만으로는 세상을 설명하기 어렵습니다. 그래서 여러 층을 쌓은 다층퍼셉트론이 등장합니다.

2. 다층퍼셉트론(MLP, Multi-Layer Perceptron)

다층퍼셉트론은 퍼셉트론을 여러 층으로 쌓은 구조입니다. 입력층, 은닉층, 출력층으로 구성됩니다.

입력층 → 은닉층 1 → 은닉층 2 → ... → 출력층

현실 데이터는 단순한 선 하나로 나눌 수 없는 경우가 많습니다. 여러 층이 있어야 점점 더 복잡한 특징을 학습할 수 있습니다.

그림 설명: 입력값은 은닉층을 거쳐 출력으로 전달됩니다. 학습할 때는 반대로 오차를 뒤에서 앞으로 전달하여 어떤 가중치를 얼마나 수정해야 할지 계산합니다.

층이 깊어질수록 어떤 일이 일어날까?

  • 앞쪽 층: 단순한 패턴 학습
  • 중간 층: 조합된 특징 학습
  • 뒤쪽 층: 최종 의사결정에 가까운 패턴 학습
핵심: 다층퍼셉트론은 “데이터를 여러 번 변형하면서 더 좋은 표현으로 바꾸는 구조”입니다.

3. 오차역전파(Backpropagation)

오차역전파는 신경망이 틀린 만큼 스스로 수정할 수 있게 해주는 핵심 알고리즘입니다.

1) 순전파(Forward) → 예측값 계산 2) 손실(Loss) 계산 → 얼마나 틀렸는지 측정 3) 역전파(Backpropagation) → 각 가중치의 책임 계산 4) 가중치 업데이트 → 틀림을 줄이는 방향으로 수정
그림 설명: 실선은 순전파, 점선은 역전파를 의미합니다. 출력에서 계산된 오차를 거꾸로 전달하면서 각 연결의 책임을 계산합니다.

쉬운 비유

시험 점수가 낮게 나왔다고 가정해보겠습니다. 단순히 “점수가 낮다”에서 끝나면 개선이 어렵습니다. 어느 과목이 문제였는지, 어느 단원이 약한지 찾아야 다음에 점수를 올릴 수 있습니다. 역전파는 바로 이 “책임 분석” 과정과 비슷합니다.

Loss가 w에 미치는 영향
= (Loss가 출력에 미치는 영향)
× (출력이 이전층에 미치는 영향)
× (이전층이 그 전층에 미치는 영향)
...
한 줄 요약: 역전파는 “틀린 원인을 뒤에서부터 거꾸로 추적하는 계산”입니다.

4. 기울기 소실 문제(Vanishing Gradient)

신경망이 깊어질수록, 뒤에서 전달된 기울기(gradient)가 앞쪽 층으로 갈 때 점점 너무 작아지는 문제가 생길 수 있습니다.

기울기는 “얼마나 수정해야 하는지” 알려주는 신호입니다. 그런데 이 신호가 너무 작아지면 앞쪽 층은 거의 학습되지 않습니다.

그래프 설명: Sigmoid와 Tanh의 미분값은 양끝에서 매우 작아집니다. 이런 작은 값들이 여러 층을 거치며 계속 곱해지면, 앞쪽 층으로 전달되는 기울기가 거의 0에 가까워집니다.

쉬운 비유

맨 뒤 사람이 전달하는 지시가 줄줄이 전달되다가, 앞사람에게 갈수록 목소리가 너무 작아져 거의 안 들리는 상황과 비슷합니다. 결국 앞쪽 사람은 무엇을 바꿔야 할지 모르게 됩니다.

문제결과
기울기가 너무 작아짐앞쪽 층이 거의 학습되지 않음
학습 속도 저하Loss가 잘 줄지 않음
깊은 모델 비효율층을 많이 쌓아도 성능 향상이 제한됨
중요: 활성화함수 선택이 중요한 이유 중 하나가 바로 기울기 소실 문제 때문입니다.

5. 기울기 소실 문제와 활성화함수

활성화함수는 단순히 출력을 바꾸는 기능만 하는 것이 아니라, 학습이 잘 되느냐를 결정하는 핵심 요소이기도 합니다.

그래프 설명: Sigmoid와 Tanh는 부드러운 S자 곡선이지만 양끝에서 평평해집니다. ReLU는 양수 구간에서 기울기를 유지해 깊은 신경망 학습에 유리합니다. Leaky ReLU는 음수 구간도 조금 통과시켜 죽은 뉴런 문제를 줄입니다.
함수특징장점단점
Sigmoid출력을 0~1 사이로 압축확률 해석 가능양끝에서 기울기 소실이 큼
Tanh출력을 -1~1 사이로 압축0 중심이라 Sigmoid보다 나은 경우 있음여전히 기울기 소실 가능
ReLU음수는 0, 양수는 그대로 통과계산 빠르고 깊은 네트워크에 유리음수 영역 뉴런이 죽을 수 있음
Leaky ReLU음수에도 작은 기울기 허용죽은 뉴런 문제 완화기울기 계수 설정 필요
핵심: 활성화함수는 “비선형성”을 주는 기능과 동시에, “학습이 잘 전달되게 하는 기능”도 함께 고려해야 합니다.

6. 경사하강법(Gradient Descent)

경사하강법은 오차를 줄이기 위해 가중치를 어느 방향으로 얼마나 움직일지 결정하는 방법입니다.

w ← w - learning_rate × gradient
그래프 설명: 현재 위치에서 기울기가 가리키는 방향의 반대로 조금씩 이동하면 Loss가 점점 작아집니다. 화살표는 가중치가 최적점 쪽으로 이동하는 모습을 보여줍니다.
학습률 상태문제
너무 큼최솟값을 지나쳐서 발산할 수 있음
너무 작음학습이 매우 느려짐
적절함안정적으로 Loss를 줄여감
정리: 역전파가 진단이라면, 경사하강법은 치료라고 볼 수 있습니다.

7. 속도와 정확도 문제를 해결하기 위한 경사하강법의 발전

기본 경사하강법은 실제로는 느리거나 흔들리거나 최적값을 잘 못 찾는 문제가 있습니다. 그래서 여러 개선 방법이 등장했습니다.

그래프 설명: Batch GD는 비교적 안정적으로 움직이고, SGD는 많이 흔들리지만 빠르게 진행합니다. Mini-Batch는 둘의 장점을 절충한 방식이라 실무에서 가장 많이 사용됩니다.
방식속도안정성특징
Batch느림안정적전체 데이터를 다 사용
SGD빠름불안정한 개씩 업데이트
Mini-Batch적당히 빠름적당히 안정적실무 표준에 가까움
Optimizer아이디어효과
Momentum이전 방향을 기억진동을 줄이고 더 빨리 이동
RMSprop기울기 크기에 따라 학습률 조정불균형한 학습 완화
AdamMomentum + RMSprop 결합속도와 안정성 모두 좋아 널리 사용
핵심: 경사하강법의 발전은 결국 더 빠르게, 더 안정적으로, 더 정확하게 학습하기 위한 노력입니다.

8. 전체 흐름을 한 번에 정리

1. 퍼셉트론: 가장 단순한 신경망 2. 다층퍼셉트론: 여러 층을 쌓아 복잡한 패턴 학습 3. 순전파: 입력을 넣어 예측값 계산 4. Loss: 얼마나 틀렸는지 측정 5. 오차역전파: 각 가중치의 책임 계산 6. 경사하강법: 가중치를 틀림이 줄어드는 방향으로 수정 7. 반복 학습: 점점 더 좋은 모델로 개선

10. 학생들에게 꼭 강조할 포인트

  • 기울기 소실 문제 때문에 깊은 신경망 학습이 어려워질 수 있다.
  • 경사하강법과 Optimizer는 속도와 정확도를 크게 좌우한다.
