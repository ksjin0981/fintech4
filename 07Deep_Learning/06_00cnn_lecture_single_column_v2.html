1. CNN이란 무엇인가

CNN은 Convolutional Neural Network, 즉 합성곱 신경망입니다. 주로 이미지, 영상, 패턴 데이터처럼 공간적인 구조를 가진 데이터를 다룰 때 강력합니다.

핵심 개념

이미지는 단순한 숫자 목록이 아니라, 가까운 픽셀끼리 의미 있는 관계를 가집니다. CNN은 이 점을 활용해서 모서리, 선, 질감, 부분 형태, 전체 형태를 단계적으로 학습합니다.

예를 들어 고양이 사진을 분류할 때 CNN은 처음부터 “고양이”를 바로 이해하는 것이 아니라,

  1. 초기 층에서 선, 경계, 방향 같은 단순 특징을 찾고
  2. 중간 층에서 눈, 귀, 털결 같은 부분 특징을 조합하고
  3. 깊은 층에서 전체적인 고양이 형태를 인식합니다.

2. 왜 완전연결층만으로는 이미지 처리가 어려운가

일반적인 완전연결층(Dense layer)은 입력의 모든 값이 다음 층의 모든 뉴런과 연결됩니다. 이미지에 이 방식을 그대로 적용하면 문제가 생깁니다.

문제 설명
파라미터 수 폭증 예를 들어 224×224×3 이미지를 그대로 펼치면 입력 차원이 매우 커져 가중치 수가 급격히 증가합니다.
공간 정보 손실 이미지를 1차원으로 펼치면 원래 가까이 있던 픽셀의 위치 관계가 약해집니다.
비효율적인 학습 같은 형태가 이미지의 다른 위치에 있어도 별개로 학습해야 할 수 있습니다.
이미지는 “어떤 값이 있느냐”도 중요하지만, 그 값이 어디에 있느냐도 중요합니다. CNN은 이 공간 정보를 더 잘 보존합니다.

3. 합성곱 연산이란 무엇인가

합성곱 연산은 작은 필터(커널)를 입력 데이터 위에서 한 칸씩 이동시키며, 각 위치의 국소 영역(local region)과 계산하는 방식입니다.

출력값 = 입력의 일부 영역 × 필터 가중치 를 원소별로 곱한 뒤 모두 더한 값

즉, 필터는 이미지 전체를 한 번에 보는 것이 아니라 작은 창(window)처럼 움직이며 특정 패턴이 있는지를 검사합니다. 

예시: 3×3 필터가 5×5 이미지 위를 이동

[입력 일부 영역]   ×   [필터]
값을 곱해서 더함  →   하나의 출력값 생성

이 과정을 여러 위치에서 반복하면
하나의 특성맵(feature map)이 만들어짐

필터가 “세로선”을 잘 찾아내도록 학습되면, 세로선이 있는 위치에서 출력값이 크게 나타날 수 있습니다.

4. 합성곱 연산을 하는 이유

4-1. 지역적 특징을 잡기 위해

이미지에서는 전체보다도 먼저 작은 부분의 패턴이 중요합니다. 예를 들어 모서리, 밝기 변화, 질감 같은 정보는 국소 영역에서 잘 드러납니다.

4-2. 파라미터 수를 줄이기 위해

완전연결층은 모든 입력과 모든 뉴런을 연결하지만, 합성곱은 작은 필터를 공유해서 사용합니다. 그래서 필요한 가중치 수가 크게 줄어듭니다.

4-3. 같은 패턴을 위치와 상관없이 찾기 위해

고양이의 눈이 이미지 왼쪽 위에 있든 오른쪽 아래에 있든, 눈이라는 패턴 자체는 같습니다. CNN은 동일한 필터를 전체 영역에 적용하므로 패턴 탐지의 재사용이 가능합니다.

4-4. 계층적으로 특징을 학습하기 위해

CNN은 낮은 수준의 특징에서 높은 수준의 특징으로 점점 추상화합니다.

학습의 계층 구조
  • 초기층: 선, 경계, 방향
  • 중간층: 질감, 모양 조각, 부분 구조
  • 깊은층: 얼굴, 바퀴, 동물의 귀 같은 복합 패턴

4-5. 계산 효율을 높이기 위해

입력 전체를 한 번에 처리하지 않고 작은 필터를 반복 적용하므로, 이미지와 같은 대규모 입력에서도 실용적으로 학습할 수 있습니다.

5. 필터·커널·특성맵 이해하기

용어 의미
필터(Filter) / 커널(Kernel) 입력 위를 이동하면서 특정 패턴을 찾는 작은 가중치 행렬
특성맵(Feature Map) 필터를 적용한 결과로 만들어지는 출력 맵
채널(Channel) RGB 이미지라면 보통 3개 채널(R, G, B)

필터가 여러 개 있으면 여러 종류의 특징을 동시에 추출할 수 있습니다. 예를 들어 어떤 필터는 가로선을, 다른 필터는 세로선을, 또 다른 필터는 곡선을 잘 찾게 학습될 수 있습니다.

중요: 필터 값은 사람이 직접 정하는 경우도 있지만, 딥러닝에서는 대부분 학습을 통해 자동으로 최적화됩니다.

6. 스트라이드와 패딩

6-1. 스트라이드(Stride)

필터가 입력 위를 이동할 때 몇 칸씩 이동할지를 의미합니다.

  • stride = 1 : 한 칸씩 이동
  • stride = 2 : 두 칸씩 이동

스트라이드가 커질수록 출력 크기는 작아지고 계산량도 줄지만, 세부 정보가 더 빨리 줄어들 수 있습니다.

6-2. 패딩(Padding)

입력 가장자리에 0 같은 값을 덧붙이는 것입니다.

  • valid : 패딩 없이 계산
  • same : 출력 크기를 어느 정도 유지하도록 패딩 적용

패딩을 사용하는 이유는 가장자리 정보가 너무 빨리 사라지는 것을 막고, 출력 크기를 조절하기 위해서입니다.

패딩이 없으면 합성곱을 반복할수록 이미지 크기가 빠르게 줄어듭니다. 그러면 중요한 경계 정보가 손실될 수 있습니다.

7. 활성화 함수와 비선형성

합성곱 뒤에는 보통 활성화 함수가 붙습니다. 가장 대표적인 것이 ReLU입니다.

ReLU(x) = max(0, x)

활성화 함수를 쓰는 이유는 모델에 비선형성을 주기 위해서입니다. 만약 선형 연산만 반복하면 깊은 신경망이라도 표현력이 제한됩니다.

쉽게 말하면
합성곱은 특징을 뽑고, 활성화 함수는 그 특징을 더 유연하게 조합할 수 있게 만듭니다.

8. Pooling의 종류

Pooling은 특성맵의 크기를 줄이면서 중요한 정보를 요약하는 연산입니다.

8-1. Max Pooling

작은 영역 안에서 가장 큰 값만 남깁니다.

강하게 반응한 특징이 있다면 그 특징을 대표값으로 보존하는 방식입니다.

8-2. Average Pooling

작은 영역 안의 값들의 평균을 사용합니다.

전체적인 경향을 부드럽게 요약하는 데 유리합니다.

8-3. Global Average Pooling

특성맵 전체를 채널별로 평균내어, 채널마다 하나의 값만 남기는 방식입니다.

마지막 분류 직전에 자주 사용되며, 파라미터 수를 줄이고 과적합을 완화하는 데 도움을 줍니다.

8-4. Global Max Pooling

채널별 전체 영역에서 가장 큰 값 하나만 남깁니다.

Pooling 종류 대표 방식 특징
Max Pooling 최댓값 선택 강한 특징 강조
Average Pooling 평균값 사용 전체 경향 반영
Global Average Pooling 채널 전체 평균 분류 직전 차원 축소에 자주 사용
Global Max Pooling 채널 전체 최댓값 가장 강한 반응만 남김

9. Pooling을 하는 이유

9-1. 특성맵 크기를 줄이기 위해

합성곱을 여러 번 적용하면 특성맵이 많아지고 계산량이 커집니다. Pooling은 크기를 줄여 계산 효율을 높입니다.

9-2. 중요한 정보만 남기기 위해

모든 픽셀이 똑같이 중요한 것은 아닙니다. Max Pooling은 특히 “강하게 반응한 특징”을 남기는 데 유용합니다.

9-3. 작은 위치 변화에 덜 민감하게 만들기 위해

물체가 이미지 안에서 약간 이동하더라도 같은 물체로 인식해야 합니다. Pooling은 위치 정보의 세밀함을 조금 줄이는 대신, 약간의 이동이나 변형에 더 강한 표현을 만들어줍니다.

9-4. 과적합을 완화하기 위해

차원이 줄어들면 모델이 지나치게 복잡해지는 것을 어느 정도 막을 수 있습니다.

다만 최근 모델들 중에는 Pooling을 줄이고 stride가 있는 convolution으로 대체하는 경우도 있습니다. 즉, Pooling은 매우 중요하지만 절대적으로 항상 필요한 것은 아닙니다.

10. CNN의 전체 흐름

입력 이미지
→ Convolution
→ Activation(ReLU)
→ Pooling
→ Convolution
→ Activation(ReLU)
→ Pooling
→ ...
→ Flatten 또는 Global Average Pooling
→ Dense Layer
→ Output Layer

10-1. Flatten이란?

2차원 또는 3차원 특성맵을 1차원 벡터로 펼치는 과정입니다. 이후 Dense layer에 연결할 수 있게 됩니다.

10-2. 출력층은 문제에 따라 달라진다

문제 유형 출력층 예시 활성화 함수
이진 분류 뉴런 1개 Sigmoid
다중 분류 클래스 수만큼 뉴런 Softmax
회귀 예측값 수만큼 뉴런 선형 또는 상황에 맞는 함수

11. 간단한 예시 코드

아래는 Keras 스타일의 매우 기본적인 CNN 예시입니다.

from tensorflow.keras import Sequential
from tensorflow.keras.layers import Input, Conv2D, MaxPool2D
from tensorflow.keras.layers import Flatten, Dense

model = Sequential([
    Input(shape=(28, 28, 1)),
    Conv2D(32, (3, 3), padding='same', activation='relu'),
    MaxPool2D((2, 2)),
    Conv2D(64, (3, 3), padding='same', activation='relu'),
    MaxPool2D((2, 2)),
    Flatten(),
    Dense(128, activation='relu'),
    Dense(10, activation='softmax')
])

코드 해석

  • Conv2D(32, (3,3)) : 3×3 필터 32개로 특징 추출
  • MaxPool2D((2,2)) : 특성맵 크기 축소
  • Flatten() : 특성맵을 1차원으로 변환
  • Dense(10, activation='softmax') : 10개 클래스 분류

12. 자주 헷갈리는 질문 정리

Q1. 필터 크기는 왜 3×3을 많이 쓰나요?

3×3은 작은 영역을 보면서도 충분한 패턴을 잡을 수 있고, 파라미터 수와 표현력의 균형이 좋아서 자주 사용됩니다.

Q2. Pooling을 꼭 해야 하나요?

전통적인 CNN에서는 매우 자주 사용되지만, 최근에는 stride convolution이나 다른 구조로 대체되기도 합니다.

Q3. 합성곱은 사람이 직접 규칙을 넣는 건가요?

아닙니다. 필터의 초기값은 보통 무작위로 시작하고, 학습 과정에서 손실함수를 줄이는 방향으로 자동 조정됩니다.

Q4. CNN은 이미지에만 쓰이나요?

주로 이미지에서 유명하지만, 시계열 데이터나 음성 신호처럼 국소 패턴이 중요한 데이터에도 사용할 수 있습니다.

  • 파라미터 수를 줄이기 위해
  • 같은 패턴을 여러 위치에서 재사용하기 위해
  • 계층적으로 특징을 학습하기 위해
Pooling의 종류
  • Max Pooling
  • Average Pooling
  • Global Average Pooling
  • Global Max Pooling
Pooling을 하는 이유
  • 특성맵 크기 축소
  • 계산량 감소
  • 중요한 정보 요약
  • 작은 위치 변화에 덜 민감
  • 과적합 완화에 도움

한 문장으로 정리하면, CNN은 이미지의 공간 구조를 살리면서 중요한 특징을 효율적으로 추출하는 신경망입니다.
