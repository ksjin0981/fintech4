2. 활성화 함수(Activation Function)

활성화 함수는 신경망이 단순 직선이 아니라 복잡한 곡선과 비선형 관계를 학습할 수 있게 만드는 함수입니다.

활성화 함수가 없으면 층을 여러 개 쌓아도 결국 전체 모델은 하나의 큰 선형식처럼 동작할 수 있습니다. 그래서 깊은 신경망의 의미가 크게 줄어듭니다.

대표 활성화 함수 비교

함수수식출력 범위주요 특징주요 사용처
Sigmoidσ(x)=1/(1+e-x)0 ~ 1확률처럼 해석 가능, 큰 |x|에서 기울기 작아짐이진분류 출력층
Tanhtanh(x)-1 ~ 10 중심이라 Sigmoid보다 학습이 조금 더 안정적예전 RNN, 일부 은닉층
ReLUmax(0, x)0 ~ ∞간단하고 빠르며 딥러닝 은닉층에서 매우 널리 사용CNN, MLP 기본 은닉층
Leaky ReLUx>0이면 x, 아니면 αx음수도 약하게 통과ReLU의 dying ReLU 완화ReLU 대안
GELU부드러운 확률적 게이트연속적작은 음수도 부드럽게 반영, Transformer에서 자주 사용BERT, GPT 계열
Softmax여러 출력을 확률 분포로 변환각 클래스 확률 합=1다중분류 출력층이미지/텍스트 분류

Sigmoid 직관

입력이 아무리 커도 출력은 1에 가까워지고, 아무리 작아도 0에 가까워집니다. 그래서 확률 출력에 적합합니다.

ReLU 직관

음수는 0으로 끊고, 양수는 그대로 통과시킵니다. 계산이 단순하고 깊은 모델에서 잘 동작합니다.

Tanh 직관

Sigmoid와 비슷하지만 출력이 -1~1이라 0 중심입니다. 그래서 일부 상황에서 더 균형 있게 학습됩니다.

3. 활성화 함수 그래프 시각화

강의에서는 그래프를 먼저 보여주면 이해가 훨씬 빨라집니다. 아래 이미지는 Python으로 생성한 그래프입니다.

Sigmoid

설명: S자 형태의 함수로, 입력값을 0~1 사이로 압축합니다. 그래서 출력값을 확률처럼 해석할 수 있어 이진분류의 출력층에서 자주 사용됩니다. 다만 양끝 구간에서는 기울기가 매우 작아져 학습이 느려질 수 있습니다.

Tanh

설명: Sigmoid와 비슷한 곡선이지만 출력 범위가 -1~1입니다. 중심이 0이기 때문에 Sigmoid보다 은닉층 학습이 조금 더 안정적인 경우가 있습니다. 하지만 역시 양끝에서는 기울기 소실 문제가 생길 수 있습니다.

ReLU

설명: 0보다 작은 값은 0으로 만들고, 0보다 큰 값은 그대로 통과시킵니다. 구조가 단순해 계산이 빠르고, 딥러닝 은닉층에서 가장 널리 사용됩니다. 다만 음수 구간 뉴런이 계속 0만 내면 죽은 뉴런 문제가 생길 수 있습니다.

Leaky ReLU

설명: ReLU와 비슷하지만 음수 구간을 완전히 0으로 막지 않고 아주 작은 기울기를 남겨둡니다. 그래서 ReLU의 dead neuron 문제를 완화하는 데 도움이 됩니다. ReLU의 대안으로 자주 소개됩니다.

GELU

설명: ReLU보다 더 부드럽게 입력을 통과시키는 활성화 함수입니다. 작은 음수값도 완전히 잘라내지 않고 확률적으로 반영하는 느낌을 가집니다. Transformer 계열 모델(BERT, GPT류)의 은닉층에서 많이 사용됩니다.

Swish

설명: 입력값에 Sigmoid를 곱한 형태로, 매우 부드러운 곡선을 만듭니다. ReLU처럼 급격하게 끊기지 않아서 학습이 더 매끄럽게 진행되는 경우가 있습니다. EfficientNet 등 일부 최신 모델에서 좋은 성능을 보인 함수입니다.

그래프를 어떻게 해석할까?

  • Sigmoid: 완만한 S자 모양이며 양끝에서 평평해져 gradient vanishing이 생기기 쉽습니다.
  • Tanh: Sigmoid보다 중심이 0이라 은닉층에서 조금 더 학습이 편한 경우가 있습니다.
  • ReLU: x<0 에서는 0, x>0 에서는 직선이라 계산이 빠르고 깊은 네트워크에 유리합니다.
  • GELU/Swish: ReLU보다 부드럽게 동작하여 Transformer 계열에서 선호되는 경우가 많습니다.

그래프를 그리는 Python 코드

import numpy as np
import matplotlib.pyplot as plt

x = np.linspace(-6, 6, 400)

sigmoid = 1 / (1 + np.exp(-x))
tanh = np.tanh(x)
relu = np.maximum(0, x)

plt.figure(figsize=(8, 5))
plt.plot(x, sigmoid, label="Sigmoid")
plt.plot(x, tanh, label="Tanh")
plt.plot(x, relu, label="ReLU")
plt.axhline(0, linewidth=1)
plt.axvline(0, linewidth=1)
plt.grid(True, alpha=0.3)
plt.legend()
plt.title("Activation Functions")
plt.show()

4. Loss 함수

Loss 함수는 모델이 얼마나 틀렸는지를 숫자로 나타냅니다. 학습의 목표는 이 Loss를 작게 만드는 것입니다.

문제 유형대표 Loss설명자주 연결되는 활성화
회귀MSE오차를 제곱하여 평균. 큰 오차에 더 민감함출력층 선형(linear)
회귀MAE절대 오차 평균. 이상치 영향이 MSE보다 덜함출력층 선형(linear)
이진 분류Binary Cross Entropy정답 0/1과 예측 확률 차이를 측정Sigmoid
다중 분류Categorical Cross Entropy여러 클래스 확률 분포와 정답 분포 차이를 측정Softmax

MSE 직관

실제값과 예측값의 차이를 제곱하므로 큰 오차를 더 강하게 벌점 줍니다.

MSE = (1/n) Σ (y - ŷ)²

Cross Entropy 직관

정답 클래스의 확률을 낮게 예측하면 큰 벌점을 줍니다. 분류에서 매우 핵심적인 Loss입니다.

BCE = - 평균[ y log(ŷ) + (1-y) log(1-ŷ) ]

Binary Cross Entropy 계산 예시 코드

import numpy as np

y_true = np.array([1, 0, 1, 1])
y_pred = np.array([0.90, 0.20, 0.60, 0.80])

# Binary Cross Entropy
bce = -np.mean(
    y_true * np.log(y_pred) +
    (1 - y_true) * np.log(1 - y_pred)
)

print("BCE =", bce)
중요: 이진분류에서는 보통 Sigmoid + BCE, 다중분류에서는 보통 Softmax + Cross Entropy 조합을 많이 사용합니다.

5. Backpropagation 설명 슬라이드

Backpropagation(역전파)은 출력에서 계산된 오차를 거꾸로 전달하여, 각 가중치가 Loss에 얼마나 영향을 줬는지 계산하는 과정입니다.

역전파를 왜 할까?

딥러닝은 가중치가 매우 많기 때문에 어떤 가중치를 얼마나 수정해야 할지 자동으로 계산해야 합니다.

핵심 아이디어

  • 먼저 순전파로 예측값 ŷ를 계산한다.
  • Loss를 계산한다.
  • Loss를 각 가중치에 대해 미분한다.
  • 기울기가 큰 방향의 반대로 가중치를 업데이트한다.

연쇄법칙(Chain Rule)이 핵심

Loss가 w에 미치는 영향
= (Loss가 출력에 미치는 영향)
× (출력이 activation에 미치는 영향)
× (activation이 선형결합 z에 미치는 영향)
× (z가 가중치 w에 미치는 영향)
역전파는 어려워 보이지만, 실제로는 “최종 오차를 앞쪽 층으로 거꾸로 나누어 책임을 배분하는 과정”으로 이해하면 됩니다.

학생에게 설명할 때의 쉬운 비유

축구팀이 졌다고 할 때, 단순히 “결과가 나빴다”에서 끝나는 것이 아니라, 수비·미드필드·공격 중 어디가 얼마나 영향을 줬는지 따져서 다음 경기 전략을 수정하는 것과 비슷합니다. 역전파는 각 층과 가중치의 책임을 계산하는 절차입니다.

6. Optimizer

Optimizer는 계산된 기울기를 이용해 가중치를 실제로 업데이트하는 방법입니다.

Optimizer핵심 아이디어장점주의점
SGD기울기 방향으로 조금씩 이동단순하고 해석 쉬움느릴 수 있고 진동 가능
Momentum이전 이동 방향을 누적골짜기에서 빠르게 이동하이퍼파라미터 조정 필요
RMSprop기울기 크기에 따라 학습률 조절학습률 자동 조정 효과설정 이해 필요
AdamMomentum + RMSprop 결합대부분 상황에서 안정적항상 최선은 아닐 수 있음
AdamWAdam + weight decay 분리Transformer에서 매우 널리 사용weight decay 이해 필요

Gradient Descent 시각화

학습률(Learning Rate)이 중요한 이유

  • 너무 크면 최소값을 지나쳐 발산할 수 있음
  • 너무 작으면 학습이 매우 느려짐
  • Optimizer는 이 이동을 더 똑똑하게 만들어주는 장치
같은 모델이라도 Optimizer와 learning rate에 따라 성능이 크게 달라질 수 있습니다.

아주 단순한 Gradient Descent 코드

# 매우 단순화한 Gradient Descent 예시
w = 3.0
lr = 0.1

for step in range(10):
    grad = 2 * (w - 1.3)   # d/dw of (w-1.3)^2
    w = w - lr * grad
    print(step, round(w, 4))

7. Optimizer 수식 설명

강의용으로는 너무 복잡한 전체 증명보다, “무엇을 저장하고 어떻게 갱신하는가”를 중심으로 설명하는 것이 효과적입니다.

1) 기본 Gradient Descent

w ← w - η ∂L/∂w
  • w: 가중치
  • η (eta): learning rate
  • ∂L/∂w: Loss를 w로 미분한 값(기울기)

2) Momentum

v_t = β v_(t-1) + (1-β) ∂L/∂w
w ← w - η v_t

이전 방향을 기억해서 관성처럼 움직입니다. 골짜기 형태의 손실함수에서 흔들림을 줄일 수 있습니다.

3) RMSprop

s_t = β s_(t-1) + (1-β) (∂L/∂w)²
w ← w - η · (∂L/∂w) / (√s_t + ε)

기울기가 큰 파라미터는 보폭을 줄이고, 작은 파라미터는 상대적으로 더 움직이게 합니다.

4) Adam

m_t = β₁ m_(t-1) + (1-β₁) ∂L/∂w
v_t = β₂ v_(t-1) + (1-β₂) (∂L/∂w)²

m̂_t = m_t / (1-β₁ᵗ)
v̂_t = v_t / (1-β₂ᵗ)

w ← w - η · m̂_t / (√v̂_t + ε)

m

기울기의 평균 방향을 저장

v

기울기의 크기 정보를 저장

결과

방향과 보폭을 모두 조절해서 안정적으로 학습

강의 포인트: Adam은 “방향 기억(Momentum)” + “보폭 자동 조절(RMSprop)”을 합친 것이라고 설명하면 학생들이 빠르게 이해합니다.

8. CNN / Transformer에서 실제 사용하는 activation

실전에서는 모델 구조에 따라 자주 쓰이는 활성화 함수가 조금 다릅니다.

CNN에서 많이 쓰는 activation

모델/구조주요 activation이유
기본 CNNReLU계산이 빠르고 이미지 모델에서 오랫동안 표준처럼 사용
ResNet 계열ReLU안정적이고 구현이 쉬움
EfficientNet 일부 변형Swish / SiLU부드러운 활성화가 성능 향상에 도움
출력층Sigmoid / Softmax이진분류 또는 다중분류 확률 출력

Transformer에서 많이 쓰는 activation

모델/구조주요 activation이유
BERTGELU부드러운 게이팅 특성으로 표현력이 좋음
GPT 계열GELU / SwiGLU 계열대규모 언어모델 FFN에서 성능상 이점
T5 계열ReLU / GLU 변형구조별 설계 철학 차이
출력층Softmax다음 토큰 확률 분포 생성

왜 CNN은 ReLU, Transformer는 GELU 계열이 많을까?

  • CNN: 계산 효율성과 안정성이 중요해서 ReLU가 오랫동안 표준 역할을 해왔습니다.
  • Transformer: 아주 깊고 큰 모델에서 부드러운 활성화가 표현력과 학습 안정성에 도움이 되어 GELU 계열이 널리 채택되었습니다.
수업에서는 이렇게 정리하면 좋습니다: “이미지 쪽 전통 CNN은 ReLU 중심, 최신 대규모 언어모델과 Transformer는 GELU 계열이 많다.”

10. 최종 정리

Activation Function

복잡한 패턴을 학습하기 위한 비선형성을 제공

Loss Function

모델이 얼마나 틀렸는지 측정하는 오차의 기준

Optimizer

오차를 줄이기 위해 가중치를 실제로 바꾸는 학습 규칙

학생용 한 문장 요약

딥러닝은 활성화 함수로 복잡한 표현을 만들고, Loss로 틀린 정도를 재고, Optimizer로 그 틀림을 줄이는 방향으로 계속 수정하는 과정입니다.

강의 마무리 멘트 예시

은닉층에서 무엇을 어떻게 통과시킬지 결정하는 것이 activation,
얼마나 틀렸는지 재는 것이 loss,
그 틀림을 줄이도록 가중치를 움직이는 것이 optimizer입니다.
그리고 이 모든 수정의 핵심 계산 절차가 backpropagation입니다.
