현실적인 음식점 20개 × 2025년 1년치 데이터 생성 설명

1) 파일 구성
- realistic_store_info_20.csv: 기본 점포 목록
- realistic_store_profile_20.csv: 점포별 패턴/변동성/메모
- realistic_loan_info_20.csv: 점포별 대출 정보
- realistic_fixed_cost_20.csv: 월별 고정비 상세
- realistic_daily_cashflow_20.csv: 일별 현금흐름
- realistic_restaurant_dataset_20stores.xlsx: 위 파일을 하나의 통합 엑셀로 정리
- summary 시트: 점포별 연간 요약

2) 이번 데이터에서 반영한 현실성 요소
- 단순 우상향/우하향 대신 점포별로 서로 다른 주간 패턴 적용
  · 오피스 상권: 평일 강세 / 주말 약세
  · 유흥가/야식형: 금·토 강세 / 변동성 큼
  · 관광지/해변형: 계절성 큼
  · 대학가형: 방학 시즌 약세
  · 신규점/턴어라운드점: 중반 이후 개선
  · 노후점/공사영향점: 특정 기간 부진
- 비 오는 날, 폭염/한파, 연휴, 급여일, 스포츠 이벤트, 바이럴, 공사, 임시휴무 등을 반영
- 대출 상환은 매일 균등 차감이 아니라 특정 납부일에 월 상환액이 빠지도록 구성
- 월 고정비(임차료/급여/보험/고정 유틸리티/고정 마케팅)는 실제 현금 유출처럼 일부 날짜에 몰아서 반영되도록 설계
  · rent/급여/보험 등은 daily_cashflow의 other_expense에 지급일 기준으로 녹아 있음
  · fixed_cost 파일은 같은 고정비의 월별 breakdown 용도임
  · 따라서 daily_cashflow의 net_cashflow에 fixed_cost_total을 다시 더 차감하면 이중차감이 됨

3) 변동성이 큰 점포 예시
- S013: 공사 영향 적자점 (6~7월 부진 + 일부 휴무)
- S015: 해변/날씨 민감형 (계절/날씨 영향 큼)
- S020: 스포츠바 이벤트형 (특정 이벤트 날짜 급등)
- S003: 유흥가 야간형 (주말 편차 큼)
- S018: 턴어라운드 프로모션형 (초반 약세, 이후 회복)

4) 기존 스키마 호환성
- 기존 CSV의 핵심 컬럼 구조는 유지
- 추가로 store_profile 파일만 별도 제공

5) 참고
- 금액 단위는 원(KRW) 기준의 synthetic data
- 날짜는 2025-01-01 ~ 2025-12-31
