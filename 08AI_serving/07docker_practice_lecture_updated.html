Docker 명령어 실습 강의안

이 강의안은 기존 자료를 바탕으로 실습 예제를 다시 구성한 버전입니다. 명령어 연습 파트는 TensorFlow GPU Jupyter 컨테이너를 중심으로 바꾸고, Compose 연습은 WordPress + MySQL 블로그 구축 예제로 다시 작성했습니다.

환경 반영: 현재 로컬 PC에서 MySQL 서버가 이미 3306 포트를 사용 중이므로, Compose 예제에서는 DB 컨테이너 포트를 외부에 공개하지 않거나, 확인용으로 공개할 때는 3307:3306을 사용하도록 구성했습니다.

1. 실습 개요와 준비

이번 실습은 Docker의 핵심 명령을 손으로 직접 연습하는 데 목적이 있습니다. 특히 GPU 이미지를 pull 하고, 컨테이너를 띄우고, 포트를 매핑하고, 로그와 상태를 확인하는 흐름을 TensorFlow Jupyter 컨테이너로 익히게 됩니다.

실습 목표
① Docker 기본 명령어 익히기
② TensorFlow GPU Jupyter 이미지를 받아 컨테이너 실행하기
③ 이미지·컨테이너·네트워크·볼륨·파일 복사를 직접 연습하기
④ Docker Compose로 WordPress + MySQL 블로그 환경을 구축하기

실습 환경

  • Windows + Docker Desktop 또는 Ubuntu/Linux + Docker Engine
  • NVIDIA GPU 사용 가능 환경이면 TensorFlow GPU 컨테이너 실습에 더 적합
  • 웹브라우저, 터미널 또는 PowerShell

사전 확인

docker --version
docker compose version

구버전 환경은 docker-compose 명령을 사용할 수 있습니다. 최신 환경은 보통 docker compose 형식입니다.

개념비유설명
이미지설치 파일 / 설계도컨테이너를 만들기 위한 재료
컨테이너실행 중인 프로그램이미지로부터 실제 실행된 인스턴스
볼륨외장 저장 공간컨테이너를 지워도 데이터 보관 가능
바인드 마운트내 폴더 바로 연결호스트 폴더를 컨테이너와 직접 연결
포트 매핑건물 호수 연결호스트 포트와 컨테이너 포트를 연결해 외부에서 접속 가능

2. 컨테이너 기본 명령어

먼저 아주 간단한 웹서버 컨테이너를 띄우며 docker run, docker ps, docker stop, docker rm 을 익힙니다. 이 단계는 뒤의 TensorFlow 컨테이너 실습을 위한 워밍업입니다.

2-1. Apache와 Nginx 실행

docker run --name apa000ex2 -d -p 8081:80 httpd
docker run --name nginx000ex3 -d -p 8082:80 nginx

2-2. 실행 중인 컨테이너 확인

docker ps
docker ps -a

2-3. 웹브라우저 확인

  • http://localhost:8081 → Apache 기본 페이지
  • http://localhost:8082 → Nginx 기본 페이지

2-4. 컨테이너 정지와 삭제

docker stop apa000ex2
docker stop nginx000ex3

docker rm apa000ex2
docker rm nginx000ex3
docker ps -a

2-5. 이벤트 보기

docker system events
실습 체크: 컨테이너를 하나 다시 띄운 뒤, 다른 터미널에서 docker system events 를 켜놓고 stop / rm 이벤트가 어떻게 찍히는지 확인해 보세요.

3. TensorFlow Docker 실습

기존 강의안의 데이터베이스 예제 대신, 이 파트는 사용자가 요청한 대로 TensorFlow GPU Jupyter 이미지를 기준으로 Docker 명령을 연습하도록 재구성한 부분입니다.

주의: 이 이미지는 GPU 지원용입니다. GPU 환경이 준비되지 않았다면 컨테이너가 실행되지 않거나 GPU를 인식하지 못할 수 있습니다. 그래도 pull / images / run / ps / logs / exec / stop / rm 같은 Docker 명령어 연습 자체는 매우 유용합니다.

3-1. 이미지 다운로드

docker pull tensorflow/tensorflow:latest-gpu-jupyter

3-2. 다운로드된 이미지 확인

docker image ls
docker image ls tensorflow/tensorflow

3-3. TensorFlow Jupyter 컨테이너 실행

Jupyter 기본 포트는 컨테이너 내부에서 8888입니다. 호스트에서는 충돌을 피하기 위해 8890 포트로 연결합니다.

docker run --name tf-gpu-jupyter -d \
  -p 8890:8888 \
  --gpus all \
  tensorflow/tensorflow:latest-gpu-jupyter

3-4. 실행 상태 확인

docker ps
docker ps -a
docker logs tf-gpu-jupyter

로그에 Jupyter 접속 URL과 토큰이 표시됩니다. 브라우저에서 http://localhost:8890 으로 접속해 확인합니다.

3-5. 컨테이너 내부 진입

docker exec -it tf-gpu-jupyter /bin/bash

3-6. 컨테이너 내부에서 TensorFlow 확인

python -c "import tensorflow as tf; print(tf.__version__)"
python -c "import tensorflow as tf; print(tf.config.list_physical_devices('GPU'))"

3-7. 정지와 재시작

docker stop tf-gpu-jupyter
docker start tf-gpu-jupyter

3-8. 삭제

docker rm -f tf-gpu-jupyter
실습 과제: 같은 이미지를 이용해 컨테이너 이름을 바꿔 하나 더 실행해 보고, 포트를 8891:8888 로 바꿨을 때 두 개의 Jupyter 컨테이너를 동시에 띄울 수 있는지 확인해 보세요.

7. 볼륨 마운트

7-1. 볼륨 기본 명령어

docker volume create test_volume
docker volume inspect test_volume
docker volume ls
docker volume rm test_volume
docker volume prune

7-2. TensorFlow 노트북 저장용 볼륨 예시

Jupyter 노트북 파일을 컨테이너 삭제 후에도 유지하려면 볼륨을 연결할 수 있습니다.

docker volume create tf_notebooks

docker run --name tf-gpu-jupyter -d \
  -p 8890:8888 \
  --gpus all \
  -v tf_notebooks:/tf/notebooks \
  tensorflow/tensorflow:latest-gpu-jupyter

7-3. 확인

docker volume inspect tf_notebooks
docker container inspect tf-gpu-jupyter

10. Docker Compose

이 파트는 사용자가 요청한 대로 WordPress + MySQL로 블로그를 만드는 예제로 다시 구성했습니다. 핵심은 로컬 MySQL이 이미 3306을 사용 중이라는 점입니다.

포트 충돌 주의: WordPress는 웹 접속용으로 8085:80 을 사용하고, MySQL 컨테이너는 WordPress 내부 통신만 필요하므로 가장 안전한 방법은 DB 포트를 아예 외부에 공개하지 않는 것입니다. DB를 외부 툴로 직접 확인해야 한다면 3307:3306 으로 매핑하면 됩니다.

10-1. compose.yaml 예제

services:
  db:
    image: mysql:8.0
    container_name: blog-mysql
    restart: always
    command: --default-authentication-plugin=mysql_native_password
    environment:
      MYSQL_ROOT_PASSWORD: rootpass123
      MYSQL_DATABASE: wordpressdb
      MYSQL_USER: wpuser
      MYSQL_PASSWORD: wppass123
    volumes:
      - blog_db_data:/var/lib/mysql
    networks:
      - blog-net
    # 로컬 MySQL이 3306 사용 중이므로 기본은 외부 포트 미공개
    # 외부에서 접속 테스트가 필요하면 아래 주석 해제
    # ports:
    #   - "3307:3306"

  wordpress:
    image: wordpress:latest
    container_name: blog-wordpress
    restart: always
    depends_on:
      - db
    ports:
      - "8085:80"
    environment:
      WORDPRESS_DB_HOST: db:3306
      WORDPRESS_DB_NAME: wordpressdb
      WORDPRESS_DB_USER: wpuser
      WORDPRESS_DB_PASSWORD: wppass123
    volumes:
      - blog_wp_data:/var/www/html
    networks:
      - blog-net

networks:
  blog-net:

volumes:
  blog_db_data:
  blog_wp_data:

10-2. 실행 명령

docker compose up -d
docker compose ps
docker compose logs

10-3. 접속 확인

  • 브라우저에서 http://localhost:8085 접속
  • WordPress 설치 화면이 나오면 블로그 생성 진행
  • DB 포트를 외부에 공개하지 않아도 WordPress와 MySQL은 같은 Compose 네트워크 안에서 통신 가능

10-4. 종료와 정리

docker compose stop
docker compose down

10-5. 데이터까지 완전히 지우는 경우

docker compose down -v

10-6. 자주 쓰는 Compose 명령

docker compose config
docker compose start
docker compose restart
docker compose exec wordpress bash
docker compose exec db mysql -u root -p
실습 과제: 블로그 제목을 Docker Compose Blog 로 만들고, 첫 글을 작성한 뒤 docker compose down 후 다시 docker compose up -d 했을 때 데이터가 유지되는지 확인해 보세요.

  • 로컬 MySQL 3306 포트와 충돌하지 않도록 설계를 조정하는 방법을 익혔다.
